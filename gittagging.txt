??? 태그란 무엇인가요? (What is a Tag?)
Git에서 **태그(tag)**는 특정 **커밋(commit)**을 위한 레이블 또는 책갈피와 같습니다.
태그는 프로젝트 기록에서 **릴리스(v1.0 또는 v2.0)**와 같이 중요한 지점을 표시하는 데 가장 자주 사용됩니다.
태그는 버전을 추적하고 팀이나 사용자들과 공유할 수 있는 간단하고 신뢰할 수 있는 방법입니다.

?? 일반적인 태그 유형 (Common Tag Types)
릴리스 (Releases): 태그를 사용하면 프로젝트가 릴리스 준비가 되었을 때를 표시하여, 나중에 해당 정확한 버전을 항상 찾을 수 있습니다.
마일스톤 (Milestones): 큰 기능이 완료되었거나 버그가 수정되었을 때와 같은 주요 이정표를 강조하기 위해 태그를 사용합니다.
배포 (Deployment): 많은 배포 도구들이 어떤 버전의 코드를 배포해야 할지 알기 위해 태그를 사용합니다.
핫픽스 (Hotfixes): 이전 버전을 수정해야 할 경우, 태그를 사용하면 올바른 코드를 체크아웃하고 패치하는 것이 쉬워집니다.

?? Lightweight 태그 생성 (Create a Lightweight Tag)
**경량 태그(Lightweight tag)**는 커밋에 대한 이름일 뿐입니다. 빠르고 간단하지만, 추가 정보를 저장하지는 않습니다.

유형 (Type),저장 정보 (Information Stored),용도 (Usage)
Annotated Tag,"작성자, 날짜, 메시지",릴리스 및 다른 사람들과 공유할 때 권장됩니다.
Lightweight Tag,커밋을 위한 단순한 이름 (추가 정보 없음),책갈피와 같은 역할입니다.

:예"  git tag v1.0

주석이 달린 태그 만들기( -a -m)
주석이 달린 태그에는 이름, 날짜, 메시지가 저장됩니다.
대부분의 용도에 권장됩니다.

:예" git tag -a v1.0 -m "Version 1.0 release"

태그 목록
저장소의 모든 태그 보기:

예 : git tag

태그 세부 정보 표시( git show)
태그와 해당 태그가 가리키는 커밋에 대한 세부 정보를 확인하세요.

예 : git show v1.0

원격으로 태그 푸시
기본적으로 태그는 로컬 컴퓨터에만 존재합니다.
다른 사람들이 귀하의 태그를 볼 수 있게 하려면 원격 저장소에 해당 태그를 푸시해야 합니다.
태그를 푸시하지 않으면 해당 태그는 본인만 볼 수 있으며, 로컬에서만 볼 수 있습니다.
원격 저장소에 단일 태그를 푸시하려면(예: 릴리스 태그를 만든 후):
예: 단일 태그 푸시
git push origin v1.0
알고 계셨나요? 커밋을 푸시해도 태그는 푸시 되지 않습니다
위에 표시된 대로 태그를 명시적으로 푸시해야 합니다.

모든 로컬 태그를 한 번에 원격으로 푸시하려면 (여러 개의 태그를 만든 경우 유용함):
예: 모든 태그 푸시
git push --tags


태그 삭제
로컬에서 태그 삭제:
예
git tag -d v1.0
원격 저장소에서 태그를 삭제합니다.
예
git push origin --delete tag v1.0

태그 업데이트 또는 교체(강제 푸시)
태그를 다른 커밋으로 이동하고 원격을 업데이트해야 하는 경우 다음을 사용합니다 --force.
예
git tag -f v1.0 <new-commit-hash>
<new-commit-hash>는 특정 커밋을 고유하게 식별하는 SHA-1 해시 값
git log를 실행하면 커밋 기록이 시간순으로 표시됩니다. 원하는 커밋의 맨 앞에 있는 commit 뒤의 긴 문자열(예: a1b2c3d4e5f67890...)을 복사하면 됩니다.
팁: 해시 값 전체를 복사할 필요는 없고, 보통 앞쪽 4~10자리 정도만 복사해도 Git이 해당 커밋을 식별할 수 있습니다.
git push --force origin v1.0

태그 지정 모범 사례
태그를 사용하여 프로젝트의 릴리스, 주요 이정표 또는 안정적인 지점을 표시하세요.
공개 또는 공유된 내용에는 항상 주석이 달린 태그 ( )를 사용하세요.-a -m
모든 테스트를 통과한 후 또는 코드를 배포/릴리스하기 전에 태그를 생성합니다.

태그가 이미 존재하는 경우? git tag -d <tagname> 명령어를 사용하여 로컬에서 태그를 삭제한 다음, 다시 생성하세요.
잘못된 태그를 푸시한 경우? 로컬과 원격 저장소에서 모두 삭제한 다음, 올바른 태그를 푸시하세요.
원격 저장소에 태그가 보이지 않는 경우? git push origin <tagname> 또는 git push --tags 명령어로 태그를 푸시해야 한다는 것을 기억하세요.
원격 저장소의 태그를 덮어쓰고 싶은 경우? git push --force origin <tagname> 명령어로 태그를 강제 푸시할 수 있지만, 주의하세요! 
이 작업은 해당 원격 저장소를 사용하는 모든 사용자에게 태그를 덮어쓰게 됩니다.
