주요 Git 용어
Repository﻿: Git이 프로젝트와 변경 이력을 추적하는 폴더입니다.
Clone﻿: 원격 저장소를 내 컴퓨터에 복제하는 행위입니다.
Stage﻿: Git에게 다음에 저장할 변경 사항을 알려 주는 단계입니다.
Commit﻿: 스테이징된 변경 사항의 스냅샷을 저장하는 것입니다.
Branch﻿: 동시에 여러 버전이나 기능을 작업할 수 있게 하는 분기입니다.
Merge﻿: 서로 다른 브랜치의 변경 사항을 합치는 작업입니다.
Pull﻿: 원격 저장소에서 최신 변경 사항을 가져오는 것입니다.
Push﻿: 내 변경 사항을 원격 저장소로 보내는 행위입니다.

스냅샷﻿은 Git에서 특정 시점에서 파일, 폴더 또는 작업 공간의 전체 상태를 찍은 사진과 같은 개념입니다.
즉, 프로젝트의 특정 버전 전체를 저장하는 것으로, 어떤 파일이 어떤 내용으로 존재했는지, 폴더 구조가 
어떻게 되어 있었는지를 모두 포함합니다. Git에서는 커밋을 할 때 이 스냅샷이 저장되며, 변경된 파일 전체의 상태를
그대로 기록하는 방식을 사용합니다. 이는 기존의 델타 방식(변경된 부분만 저장)과 대비되는 개념으로, 변경 사항이 있을 때마다
전체 파일 상태를 저장하지만, 중복된 데이터는 참조하는 방식으로 효율성을 높입니다. 따라서 스냅샷은 버전 관리를 위한 근본적인
데이터 단위이며, 프로그래머가 시간에 따른 코드 변화를 추적하고 관리하는 데 필수적인 개념입니다

Git 사용법
Git을 폴더에 초기화하면 Repository﻿가 생성되어 해당 폴더의 변경 사항을 추적합니다.
Git은 변경 이력을 관리하기 위해 숨겨진 폴더를 생성하여 작업을 기록합니다.
파일이 변경되거나 추가, 삭제되면 이를 modified﻿ 상태라고 합니다.
원하는 변경된 파일을 선택하여 Stage﻿ 상태로 만듭니다.
스테이징된 파일들을 Commit﻿하면 Git이 해당 시점의 변경 사항에 대한 영구적인 snapshot﻿을 저장합니다.
Git을 통해 모든 커밋 이력을 열람할 수 있고, 이전 커밋으로 언제든지 되돌아갈 수 있습니다.
Git은 모든 커밋마다 변경 파일을 별도로 저장하는 것이 아니라, 변경된 부분만 추적하여 효율적으로 관리합니다.

왜 Git인가?
개발자의 70% 이상이 Git을 사용합니다!
개발자들은 전 세계 어디에서나 함께 작업할 수 있습니다.
개발자는 프로젝트의 전체 내역을 볼 수 있습니다.
개발자는 프로젝트의 이전 버전으로 돌아갈 수 있습니다.

대부분의 Git 작업(스테이징, 커밋, 기록 보기 등)은 사용자 컴퓨터에서 수행됩니다.
Push 와 Pull로만  GitHub, GitLab 또는 Bitbucket과 같은 원격 서버와 상호 작용하여 변경 사항을 업로드하거나 다운로드합니다.

GitHub
GitHub는 Git 저장소를 호스팅하는 웹 기반 플랫폼으로, 협업, 프로젝트 관리, 코드 리뷰 등 다양한 부가 기능을 제공합니다. 
GitHub는 전 세계에서 가장 큰 소스 코드 호스팅 서비스이며, 2018년부터 마이크로소프트가 소유하고 있습니다. 
Git은 인터넷 연결 없이도 로컬에서 작동하지만, GitHub는 클라우드 기반으로 언제 어디서나 코드에 접근하고 팀원들과 공동 작업할 수 있게 합니다. 
즉, Git은 코드 버전 관리를 위한 도구이고, GitHub는 이 도구를 사용하여 협업과 소스 코드 저장을 편리하게 만든 플랫폼입니다
