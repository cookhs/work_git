Git Staging Environment
스테이징 영역은 Git에서 변경한 파일들 중에서 커밋에 포함할 내용을 임시로 보관하는 중간 공간입니다
스테이징 영역 역할
작업 디렉터리(Working Directory)에서 수정한 파일 중 원하는 파일만 선택해 스테이징 영역에 올립니다.
스테이징 영역에 올려진 파일들만 다음 커밋에 반영됩니다.
이렇게 하면 커밋에 포함할 변경 사항을 세밀하게 제어할 수 있습니다.
주요 명령어
git add <file>: 특정 파일을 스테이징 영역에 추가(스테이지)
git add --all 또는 git add -A: 모든 변경 사항을 스테이징 영역에 추가
git status: 현재 스테이징 상태와 작업 디렉터리 상태 확인
git restore --staged <file>: 스테이징 영역에서 파일을 제거하여 언스테이지
즉, 스테이징 영역은 변경 내용을 커밋하기 전에 정리하는 '대기실' 같은 곳이며, 이곳에 담긴 변경 내용만이 프로젝트 히스토리에 기록됩니다

Stage a File with git add
git add index.html
git status
출력은 
On branch master
No commits yet
Changes to be committed:
  (use "git restore --staged ..." to unstage)
    new file: index.html

Stage Multiple Files (git add --all, git add -A)
can stage all changes (new, modified, and deleted files) at once:
git add --all
git add -A

Check Staged Files with git status
git status
